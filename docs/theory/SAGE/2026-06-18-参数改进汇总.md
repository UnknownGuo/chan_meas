## 共识 vs 分歧

| 参数 | 三方共识 | 具体数值上的分歧 |
|---|---|---|
| **弱径剔除阈值** | 共识：应改为"噪声地板/局部背景 + 自适应门限"，而非"相对最强径固定dB"。OS-CFAR 或 FCME（前向连续均值剔除）是被反复提及的具体算法 | Perplexity: 地板+6~10dB；Gemini: FCME 默认10%最低能量样本估纯噪声集，$P_{fa}=10^{-4}$；ChatGPT: 不同文献从+6dB到+15dB都有，没有通用值 |
| **相对最强径动态范围** | 三方都认为-15dB偏紧，应放宽 | Perplexity给-20~-25dB（引Mota/TUM）；Gemini给"3-10倍瑞利"侧面佐证远径可信；ChatGPT不给具体数字，强调应由噪声地板决定而非固定相对值 |
| **SAGE迭代次数** | 共识：5-10次足够，应配收敛准则而非纯固定次数 | Perplexity/Gemini都引用Fleury(1999)"~10次收敛"；Gemini额外给出"15-20次硬上限+1e-3似然相对变化"双轨机制；ChatGPT引用Yin等"5次"个案 |
| **径数停止准则** | 共识：能量覆盖率（90%）是经验性、场景敏感的弱准则，更推荐MDL/BIC或增量GLRT/PFA似然阈值 | 三方都认同覆盖率法"在LOS场景过早停，在NLOS场景偏晚"这一缺陷，但都承认MDL/AIC在V2V里"未被系统验证"——这是公认的开放问题 |
| **最小延迟间隔** | 共识：不是绑定$1/B$瑞利分辨率本身，而应绑定**系统脉冲有效主瓣宽度**（与已有的`pulse_half_width_bins`概念一致） | Perplexity给0.5/B~1/B；Gemini给出和SNR/动态范围联动的范围（强LOS区0.2~0.3/B，深衰落区0.8~1/B）——这个"按SNR自适应间隔"的提法只有Gemini单独给出，需谨慎对待 |
| **多普勒搜索半宽** | 共识：应按$f_{D,max}=2f_cv_{max}/c$ + 余量计算，且必须用**15序列平均后的有效采样间隔**而非原始序列间隔 | ChatGPT和Gemini都强调15序列硬件平均会压缩可用多普勒区间（Gemini算出了具体的混叠/Sinc滚降机制），这是V2V动态数据里值得重点核实的硬件约束 |

## 已落地的改动

只动了"弱径剔除阈值"这一项，风险最低、收益最明确，其余（迭代次数收敛准则、MDL/BIC径数停止、最小延迟间隔绑定脈衝宽度、多普勒搜索半宽按有效采样间隔）暂未实施。

**`src/signal/sage_adaptive.py:312-329`**：

1. 剔除判据的参考基准从"相对最强径"换成"相对窗口噪声地板"：
   - 旧：`rel_power_db = item.power_db - strongest_power_db`，`rel_power_db < -15.0` 触发剔除
   - 新：`noise_floor_db = 10*log10(median(robust_delay_power))`（整窗噪声地板），
     `margin_above_noise_db = item.power_db - noise_floor_db`，`margin_above_noise_db < weak_prune_noise_margin_db` 触发剔除
2. 新增可调参数 `weak_prune_noise_margin_db: float = 8.0`（默认取文献6~10dB推荐区间中点），近距强LOS场景建议调大，远距强散射场景建议调小
3. 局部突出度判据（`local_prominence_db < 3.0`）不变，剔除条件仍是两者 **AND**
4. 删除了不再使用的 `strongest_power_db` 死代码

**两个判据的关系**（用合成数据验证过，见 `tmp/explain_weak_path_pruning.png`）：局部突出度负责抓"形状上不像独立峰"的径（容易被强邻径的旁瓣/裙摆污染局部地板），margin负责抓"绝对功率上接近噪声"的径；两者AND意味着只有"局部不突出 且 绝对功率也弱"才会被删——比单独用任一判据更保守。新逻辑下，"贴着强径旁瓣、局部不突出但绝对功率离噪声地板还有余量"的径会被保留，这点和旧版（用相对最强径做第二判据）的行为不同。

## 尚未实施、留作后续

- SAGE迭代次数固定值 → 收敛准则（似然相对变化<0.1%或参数变化阈值）
- PDP能量覆盖率90% → MDL/BIC或PFA似然阈值做主准则
- 最小延迟间隔绑定系统脈衝有效主瓣宽度，而非脱离系统响应的固定常数
- 多普勒搜索半宽按"15序列平均后的有效输出间隔"重新计算（`delay_doppler_sage.py:616` 处的重复剔除逻辑，-12dB无开关保护，也未同步修改）
