## 1. 背景与目标

### 背景

当前系统（洛阳测量，10ms 帧周期）收发两端各自使用独立 TCXO 晶振，没有共享参考时钟。由于两端振荡器频率存在偏差（Carrier Frequency Offset, CFO），每帧 CIR 主路径的相位会以固定速率旋转。这使得对多帧 IQ 直接做**相干平均**时，各帧 phasor 方向不一致，产生相消，SNR 增益远低于理论值 `10 log₁₀(N)` dB。

### 目标

1. **定量估计 CFO**：利用背靠背（B2B）校准文件 `接收数据帧_20251222142759088.bin` 计算 CFO（Hz）。  
   - 输出图1：CFO 随帧序号变化（相邻帧估计、各帧与第 0 帧累积相位两种视角）。

1. **可视化 CFO 对相干平均的损害**：对校准数据，分别用 前1、2、3、5、10、15 帧做相干平均（直接对复数 IQ 叠加再取模），在 PDP 图上对比，并标注主径与噪底的动态范围。

2. **对 OTA 数据做同样展示**（补充参考）：从 `接收数据帧_1.bin` 或 `接收数据帧_3.bin` 随机取一个 snapshot，同样绘制不同帧数平均后的 PDP，标注 snapshot 序号与动态范围。

## 4. CFO 估计方法

### 4.1 思路

B2B 校准场景下信道固定，每帧 CIR 主路径的复数值为：

```
CIR_k[peak_bin] = A · exp(j · (2π · Δf · k · T_frame + φ₀))
```

其中 `Δf` 为 CFO（Hz）。

相邻帧相位差：
```
Δφ_k = angle( CIR_{k+1}[peak] · conj(CIR_k[peak]) )
CFO  = Δφ_k / (2π · T_frame)          [Hz]
```

累积相位（以第 0 帧为参考）：
```
φ_k = angle( CIR_k[peak] · conj(CIR_0[peak]) )
CFO_cumulative_k = φ_k / (2π · k · T_frame)
```

### 4.2 peak_bin 的确定

对 B2B 文件，对所有帧的 |CIR| 取平均，找幅度最大的 bin，作为后续所有帧的固定 peak_bin。

### 4.3 B2B 文件 IQ 处理

B2B 文件（`142759088.bin`）与 OTA 文件格式相同（p=15），每帧 15 组 LFM：
- 将 15×1024 IQ reshape 为 `(1024, 15)`，**去掉第 1 列**，对第 2–15 列共 **14 组 IQ 做复数相干平均**，得到 1024 点复数 IQ
- 将平均后的 IQ 按**单帧处理流程**：先复制三份（长度 3×1024），再做滑动相关，得到单帧 CIR（长度 1024）
- 从各帧复数 CIR 中取 peak_bin 的复数值，用于相位跟踪

此流程与 OTA 帧内平均一致，无需 zero-pad。

## 5. 平均效果展示方法

### 5.1 相干平均（体现 CFO 损害）

对校准数据，固定起始帧 k=0，分别对 N=1,2,3,5,10,15 帧做相干平均：

```
CIR_avg_N = (1/N) · Σ_{k=0}^{N-1} CIR_k        # 复数直接相加
PDP_avg_N = 20·log₁₀(|CIR_avg_N|)               # dB
```

**图1：CFO 时变图（校准文件）**

**图2：相干平均效果对比（校准文件）**

- 横轴：delay bin（或 ns）

- 双箭头标注：N=1 时的动态范围（主径峰值 − 噪底），记录具体数值

## 7. 待确认问题

> 以下问题均已确认，无悬留项。

1. ✅ **B2B 帧内平均方式**：对 15 组 IQ 做复数相干平均；平均后的 IQ 再按单帧处理流程（先复制三份，再滑动相关）得到 PDP。复数相位信息完整保留，可直接用于 CFO 相位跟踪。

2. ✅ **CFO 量级预期**：中心频率 3.6 GHz，TCXO ±2.5 ppm，理论最大 CFO ≤ 9 kHz。注意帧间相位差法可观测范围仅 ±50 Hz（±1/(2·T_frame)），若实际 CFO 超出此范围将发生相位包裹，需在实现中做 unwrap 或借助累积相位斜率法判断。

3. ✅ **OTA 主径 peak_bin 定义**：固定使用 snapshot 第 0 帧的 peak_bin，不随帧漂移。

4. ✅ **PDP 纵轴单位**：B2B 与 OTA 图均使用归一化 dB（相对最大值），两图纵轴统一，方便对比。
