## 核心模块逻辑

### 1. config/local.py

- 被`.gitignore`排除，每台机器独立配置

### 2. src/paths.py

**功能**: 统一管理项目所有路径

**执行流程**:
1. 从`config/local`导入外部数据根目录
2. 自动推断项目根目录（当前文件的上上级目录）
3. 定义处理后数据的存放路径（项目内部的`data/`目录）

**关键路径**:
- `PROJECT_ROOT`: 项目根目录
- `RAW_MEA_DIR/RAW_CALI_DIR`: 外部原始数据（Windows数据盘挂载点）
- `CALIB_B2B_DIR/CALIB_OTA_DIR`: 校准结果输出目录

### 3. src/io/bin_reader.py

**功能**: 滑动相关器信道数据读取与处理

#### 核心常量
- 采样率100MHz，带宽50MHz，每帧4132字节
- IQ数据1024点，从第32字节开始
- GPS信息：经纬度（DDMM格式）、高度、时分秒

#### 执行流程

**帧加载**:
1. 支持单文件或整个目录批量加载
2. 可按需限制最大帧数防止内存溢出
3. 将字节流重塑为(n_frames, 4132)数组

**IQ解析**:
1. 从每帧提取4096字节IQ数据
2. 大端序解码16位有符号数（高字节在前）
3. 归一化到[-1,1]范围，组成复数IQ

**GPS解析**:
1. 提取经纬度字节（4字节大端有符号整数）
2. DDMM格式转换为度（度+分/60）
3. 提取高度（0.1dm分辨率）和时分秒

**滑动相关**:
1. 每帧去除直流偏移
2. 信号补零扩展3倍避免循环卷积
3. FFT频域相乘实现快速相关
4. 提取中央1024点作为信道冲激响应(CIR)

#### B2B诊断（替代错误CFO估计）

**问题**: 原函数将B2B峰值位置(~1300ns)误认为频偏，实际这是硬件处理时延

**正确理解**:
- B2B峰值反映ADC+滤波器的群时延
- TCXO晶振精度±2.5ppm，真实频偏≤3.5kHz
- 原估计3.2MHz(2300ppm)在物理上不可能

**诊断内容**:
1. 对所有帧CIR求平均得功率时延谱
2. 找最大功率对应的bin位置和时延
3. 计算峰值相对于噪声底的动态范围(dB)

#### 校准向量计算

**流程**:
1. 加载B2B文件，生成CIR
2. **幅度平均**（推荐）:
   - 每帧CIR做FFT得频域响应
   - 对所有帧的幅度求平均（对独立TCXO相位漂移免疫）
   - 使用第0帧相位保留硬件群时延
3. **相干平均**（不推荐用于独立振荡器）:
   - 直接对复数CIR平均
   - 独立TCXO每帧相位旋转~440弧度，相干平均导致抵消

**为什么幅度平均？**
- 独立TCXO频偏~7kHz，10ms帧周期内相位漂移大
- 相干平均要求相位一致，独立振荡器无法满足
- 幅度平均只关心能量分布，不受相位影响

#### 平均技术对比

**相干平均**:
- 直接对复数求平均
- SNR增益: 10·log₁₀(N) dB
- **前提**: 必须共享时钟源，频偏<50Hz

**非相干平均（功率）**:
- 对|CIR|²求平均
- SNR增益: 5·log₁₀(N) dB（相干的一半）
- **优点**: 对频偏免疫，适用于独立TCXO

#### 频响校准

**校准原理**:
- B2B测量: H_sys × H_att（系统×衰减器）
- 场测量: H_sys × H_PA × G_ant × H_channel × G_ant
- 校准后: H_PA × G_ant² × H_channel / H_att

**校正步骤**:
1. 对CIR做FFT转换到频域
2. 逐频点除以校准向量
3. 补偿衰减器、PA增益、天线增益
4. IFFT回到时域得校准后CIR

**PA/天线增益来源**:
- PA增益: ZHL-2W-63-S+数据手册插值
- 天线增益: MA802P全向天线数据手册插值
- 也可传入固定值或自定义数组

### 4. src/io/bin_reader_luoyang.py

**功能**: 读取洛阳10ms帧格式.bin文件

**帧格式差异**:
- 帧长61476字节（非4132字节）
- 每帧15个LFM序列（p=15）
- IQ从第32字节开始
- 字节序: [Q_hi, Q_lo, I_hi, I_lo]

**执行流程**:
1. 读取整个文件为字节数组
2. 验证帧长度（从第5-6字节解析）
3. 重塑为(n_frames, frame_len)数组
4. 解析IQ数据为(n_frames, 15, 1024)复数数组

**应用场景**:
- 洛阳测试数据使用此格式
- 每帧包含15个重复LFM，可做帧内平均

### 5. src/calibration/cfo_estimator.py

**功能**: CFO估计（载波频偏）

#### LFM匹配滤波器构造
1. 生成up-chirp LFM信号（相位πn²/U）
2. 复制3倍并取共轭FFT作为匹配滤波器
3. 用于后续滑动相关

#### CIR生成
1. 去除直流偏移
2. 信号补零扩展3倍
3. FFT频域相乘匹配滤波器
4. IFFT后提取中央1024点

#### CFO估计方法

**方法1: 相邻帧相位差**
- 计算连续两帧主导径的相位差: angle(CIR[k] × conj(CIR[k-1]))
- CFO = Δφ / (2π × 帧周期)
- 可观测范围: ±50Hz（帧周期10ms）
- 超出范围会相位卷绕

**方法2: 累积相位线性拟合**
1. 计算每帧相对于第0帧的累积相位
2. 相位展开(unwrap)处理卷绕
3. 对累积相位-时间做线性拟合
4. 斜率即为全局CFO估计
- 优点: 对噪声更鲁棒，容忍相位卷绕

#### 统计输出
- 均值、标准差、最小值、最大值
- 用于评估CFO稳定性和幅度
